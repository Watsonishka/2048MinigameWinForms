# 🧩 2048

Это классическая WinForms-реализация игры **2048**, где нужно собирать плитки с одинаковыми числами, объединяя их.

## 🏗 Архитектура

Проект разделен на несколько частей:

- **ClassLibrary** — бизнес-логика 
  
  ![кл](https://github.com/user-attachments/assets/04c350dc-2e9e-4ba6-b53c-33b50b2608fb)

- **WinFormsApp** — графический интерфейс
 
![вф](https://github.com/user-attachments/assets/1d5a8072-50a6-4023-87cb-fc98c6d11359)

В WinForms приложение построено на переключении нескольких окон (Form). Каждое окно отвечает за свой этап: главное меню, игровое поле, окно с лучшими результатами, режим игры, правила. Переход между окнами осуществляется через открытие новых форм и скрытие текущих.

## 🚀 Возможности

- 👤 Регистрация пользователя

![рег](https://github.com/user-attachments/assets/fc291173-58ed-4597-9102-476024c0caca)

Перед началом игры игрок вводит своё имя. Разрешены только буквы русского алфавита, пробел, тире и апостроф. Программа обрабатывает пользовательский ввод, заменяя заглавные буквы на строчные (кроме первой буквы каждого слова), убирает лишние знаки, оставляя  самый первый символ.

- 🎮 Выбор игрового поля
  
![поле](https://github.com/user-attachments/assets/b4e5b727-4c78-4c88-8dc8-2d4e210378a8)

Доступен выбор размера игрового поля: 4×4, 5×5, 6×6 и 7×7. Размер ячеек автоматически подстраивается под экран.

- ☰ Наличие главного меню
  
![меню](https://github.com/user-attachments/assets/2fcc6047-04d3-47b1-9c87-ad7bc99b0194)
  
Расположено в верхней части окна, выполнено в виде контекстного (выпадающего) меню.

- 🧠 Игровой процесс
  
![игра](https://github.com/user-attachments/assets/5b8a511f-8359-47ec-ad22-55095d8ccef9)

На поле каждый ход рандомно генерируется на свободной ячейке значение 2 или 4 (Двойка выпадает с 80% вероятностью). Управление осуществляется клавишами-стрелками. При каждом ходе все плитки сдвигаются в выбранном направлении, а одинаковые соседние объединяются в одну с удвоением значения. Также в игре предусмотрены система рекордов. Лучший результат сохраняется в JSON-файл. При запуске программа читает рекорд из файла и отображает его на экране. В случае побития рекорд автоматически обновляется и перезаписывается в файл.

- 🔄 Возможность начать новую игру (рестарт или выбор нового режима (выбор нового размера поля)
  
![рестарт](https://github.com/user-attachments/assets/22ebc696-b70f-4cc3-afe7-34c7a335a43f)

 При начале новой игры предыдущий прогресс сбрасывается, но лучший результат сохраняется до тех пор, пока не будет побит.
 
- 📖 Правила игры

<img width="398" height="616" alt="image" src="https://github.com/user-attachments/assets/08496874-8a15-4f4d-9a9c-ca5cedc962df" />

Предусмотрены правила для тех, кто не знаком с игрой. Оно вызывается из главного меню и содержит пошаговое объяснение механики игры, управления и цели. Это делает игру доступной для всех пользователей, независимо от опыта.

- 🏅 Таблица предыдущих результатов игры

![рекорды](https://github.com/user-attachments/assets/f910143f-b4dd-4ac2-9fb9-85abda53eed8)

Все предыдущие игровые сессии записываются в JSON-файл и читаются оттуда же. При запуске программа загружает сохранённые результаты,.

- 💾 Сохранение лучшего результов в JSON и чтение рекорда игры
  
<img width="1035" height="511" alt="image" src="https://github.com/user-attachments/assets/af6b056e-f8a6-4f8d-a687-ebc8cb797866" />

<img width="536" height="312" alt="image" src="https://github.com/user-attachments/assets/426f877f-3e9b-4551-8598-6772419710e0" />

Лучший результат загружается из JSON-файла с помощью метода GetRecordScore(), который перебирает всех пользователей в истории игр и возвращает максимальное значение счёта. Если история пуста, возвращается 0.

## ⚙️ Технологии

- C#
- .NET
- WinForms
- JSON (Newtonsoft.Json)

## 💡 Особенности реализации

- Реализована полная логика перемещения и объединения плиток
- Визуализация игрового поля с цветовой индикацией плиток
- Обработка нажатий клавиш
- Сохранение лучшего результата между сессиями
- Разделение бизнес-логики и UI
